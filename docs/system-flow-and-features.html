1. New features (highlighted)

These capabilities were added around the Smart Goal Plan initiative and full database seeding.

New

Smart nutrition & BMR (Coach → Client)

  • BMR & TDEE — Mifflin–St Jeor BMR from height, weight, date of birth, and gender; TDEE from activity level (Sedentary → Very active).
  • Activity level — Stored on the client profile; drives TDEE.
  • Auto-fill macros — One-click suggestion for daily calories, protein, carbs, and fat from goal + weight + TDEE (lib/bmr.ts).
  • Carbs & fat (grams) — Coach can edit and save; client dashboard shows full macro breakdown with protein.
  • Workout recommendation card — For selected objectives (e.g. weight loss, muscle gain), shows suggested workout type, frequency, and intensity.

Where: Coach → My Clients → open a client → profile / nutrition section.

New

Client goals are coach/admin only

  • Clients cannot add or remove their own workout goals via the API or UI.
  • Goals page is view-only with a short notice; progress updates and “View workouts” remain.

Where: Client → My Workout Goals; API POST /api/client/goals and DELETE …/goals/:id return 403.

New

Admin workouts: filter by goal

  • Toolbar Goal dropdown: all goals, no goal linked, or a specific workout goal.
  • API returns each workout’s linked goals for filtering.

Where: Admin → Workouts.

New

Full relational seed

  • npm run seed resets demo data and recreates all modules with valid foreign keys: users, profiles, memberships, renewals, equipment, exercises, workouts, media, equipment links, workout goals, goal–workout links, client goals, updates, progress, assignments, schedules, attendance, payments, meal plans, notifications.
  • Extra account: pending@croscal.local (INACTIVE client) for admin approval testing.

Warning: Seed clears application tables; use only on dev/local databases.

2. End-to-end system flow (start → end)

Phase A — Entry & authentication

1. User opens the app root / → redirect to /auth/sign-in.
2. New gym member: Register as a client /auth/register → submits profile → account is INACTIVE until an admin approves.
3. Returning user: email + password → NextAuth session created.
4. After login, redirect by role: Admin /admin/analytics · Coach /coach/dashboard · Client /client/dashboard.

Phase B — Client journey (typical)

5. Client lands on Dashboard — membership summary, attendance chart, QR for check-in, coach plan card (when coach + data exist).
6. Memberships — choose plan (Basic / Premium). Premium enables coach assignment and coach-managed experience.
7. Payments — admin records payment; when status is completed, membership becomes active.
8. With active membership, sidebar unlocks: Goals, Meal plan, Schedules, Attendance (plus payments).
9. Check-in — show QR on dashboard (or check in from a scheduled session). Coach/admin scans QR or records manual attendance.
10. Workouts — follow assigned programs linked to coach-set goals.
11. Goals — view assigned goals, progress bars, post progress notes for coach. (Adding/removing goals: coach/admin only — new.)
12. Meal plan — read-only text from coach.
13. Schedules / Attendance — upcoming sessions and history.

Phase C — Coach journey (typical)

14. My Dashboard — counts and charts for assigned clients and sessions.
15. My Clients — open client drawer: profile, nutrition & smart plan (BMR, TDEE, auto macros, recommendations — new), assign catalog goals, meal plan, workouts, log session progress.
16. Schedules — create or edit sessions (day/time for clients).

Phase D — Admin journey (typical)

17. Analytics — members, active memberships, today’s attendance, revenue charts.
18. Accounts — approve/reject registrations; manage users.
19. Memberships — define products; Client memberships — sell/assign; Coach assignments — link premium clients to coaches.
20. Payments & Renewals — record money movement and expiring memberships.
21. Workout goals & Workouts — catalog goals, link workouts to goals by day; filter workout list by goal (new).
22. Schedules & Attendance — facility calendar and check-in monitoring.
23. Equipment — inventory for workouts.

Phase E — Exit

24. Sidebar footer → user menu → Log out → session ends → /auth/sign-in.
25. Protected routes (/admin/*, /coach/*, /client/*) redirect to sign-in if unauthenticated or wrong role.
